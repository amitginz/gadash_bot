📡 תיעוד REST API

כל הנתיבים דורשים session פעיל (כניסה דרך /login). בקשות POST/PATCH חייבות לכלול את ה-CSRF token בכותרת X-CSRFToken.

GET /api/entries

מחזיר רשימת רשומות עבודה בפורמט JSON. תומך בכל פרמטרי הסינון של הדף הראשי.

Query Parameters
פרמטרסוגתיאורדוגמה
clientstringסינון לפי שם לקוח (חיפוש חלקי)?client=כרמל
date_fromYYYY-MM-DDמתאריך?date_from=2025-01-01
date_toYYYY-MM-DDעד תאריך?date_to=2025-06-30
taskstringסוג עבודה (חריש/ריסוס/קציר/דיסוק/אחר)?task=חריש
תגובה לדוגמה
[
  {
    "שם לקוח": "כרמל חקלאות",
    "תאריך": "2025-06-15",
    "עבודה": "חריש",
    "שם חלקה": "חלקה ב",
    "כמות": "30 דונם",
    "כלי": "טרקטור",
    "מפעיל": "יוסי כהן",
    "הערות": "",
    "מזין": "Web"
  }
]
דוגמת קריאה (JavaScript)
const res = await fetch('/api/entries?client=כרמל&date_from=2025-01-01');
const data = await res.json();
console.log(data.length, 'רשומות');
PATCH /api/entries/<row_id>

עדכון שדה בודד ברשומה קיימת. משמש לעריכה מוטבעת (double-click בטבלה).

Request Headers
כותרתערך
Content-Typeapplication/json
X-CSRFTokenערך ה-CSRF מ-<meta name="csrf-token">
Request Body
שדהסוגתיאור
fieldstringשם העמודה לעדכון (אחד מ: שם לקוח, תאריך, עבודה, שם חלקה, כמות, כלי, מפעיל, הערות)
valuestringהערך החדש
גוף הבקשה (JSON)
{ "field": "כמות", "value": "45 דונם" }
תגובה מוצלחת
{ "ok": true, "row_id": 42, "field": "כמות", "value": "45 דונם" }
תגובות שגיאה
קודסיבה
400שדה לא תקין (מחוץ לרשימה המותרת)
403CSRF token חסר או שגוי
404row_id לא קיים
דוגמת קריאה (JavaScript)
const csrf = document.querySelector('meta[name="csrf-token"]').content;
const res  = await fetch('/api/entries/42', {
  method: 'PATCH',
  headers: { 'Content-Type': 'application/json', 'X-CSRFToken': csrf },
  body: JSON.stringify({ field: 'כמות', value: '45 דונם' })
});
const data = await res.json();
if (data.ok) console.log('עודכן!');
